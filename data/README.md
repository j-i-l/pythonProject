# Data Directory

This directory contains the datasets utilized and generated by this project.
To ensure reproducibility and data integrity, strict separation of data stages is enforced.

## Directory Structure

* **`raw/`**:  
  Original, immutable data dumps. **Files in this directory must never be edited.**
  The result of all processing must reside in the `interim/` or `final/` folder.
* **`interim/`**:  
  Intermediate data that has been transformed or partially cleaned, but is not yet ready for final analysis.
* **`final/`**:  
  Cleaned, processed data ready for modeling, analysis, or publication.

## Version Control

By default, all data files (e.g., `.csv`, `.json`, `.h5`, `.tif`) are ignored by Git to prevent repository bloat and the accidental leakage of sensitive information.
Only this `README.md`, small datasets (< 100MB) and explicitly tracked metadata files should be committed.



## Advanced Data Tracking

For large or frequently updated datasets, advanced tracking mechanisms are highly recommended:

* **Git Submodules & Git LFS**:  
  Dedicated data repositories utilizing Git LFS (Large File Storage) can be linked to this project as `git submodules`.
  This allows a specific, version-controlled state of an external dataset to be mounted directly within the `data/raw/` directory without polluting the primary analysis repository's history.
* **Data Version Control (DVC)**:  
  DVC can be employed to track massive datasets and model artifacts.
  It stores the actual data in remote storage (e.g., OpenStack Swift, AWS S3, institutional clusters) while committing lightweight `.dvc` tracking files to Git.
  DVC can be used independently or integrated with Git submodules (where the submodule provides the DVC metadata, and a subsequent `dvc pull` command retrieves the actual files).
